Setup Guide

  1. Download Node.js
  2. Clone Repository.
  3. Navigate into server folder and run the below commands. This installs the dependencies listed in package.json.
    Dependencies:
    npm install express --save
npm install pg --save
npm install cors --save
npm install nodemon --save
npm install @react-oauth/google
npm install jwt-decode
npm install react-icons --save
    • express: Framework for creating creating REST API, handles HTTP requests.
    • pg: connects to postgress.
    • cors: allows HTTP connections between different domains.
    • nodemon: automtically restarts server when saving new changes.
  4. Start the Node.js server with "node index.js" or "nodemon index.js" to utilize auto restart.
  5. Open a second terminal and navigate into the client folder.
  6. Run all the commands below again to install the dependencies listed in package.json.
    npm install react --save
npm install bootstrap --save
npm install react-router --save
npm install google-map-react --save
npm install @react-oauth/google
npm install jwt-decode
npm i axios --save
npm install react-i18next i18next --save
npm install react-icons --save
npm install flowbite-react --save
npm install react-virtualized --save
npm install react-datepicker
    Dependencies:
    • react: javascript library.
    • bootstrap: css framework.
    • router:
  7. Start the react app with "npm start" and a browser window should open. When prompted if you would like to use the same port, choose yes.

Deployment

React App: https://kungfuteaproject3.onrender.com
Node.js Server: https://kungfuteaserver09r.onrender.com

Bug: Anchor links <a href="URL"> do not work during deployment. They send a request to the web server but Render.com is not configured to handle them. Use React Router <Link to="URL"> instead, it handles routing in the browser and does not contact the web server.

JSDoc

  • Install it globally with npm install -g jsdoc check installation with jsdoc -v.
  • Pull from main and make sure you have the configuration file jsdoc.json, it should be in the Project3 folder.
  • Create JSDoc comments the same way as JavaDoc, PopularDrink.js has an example
  • Generate html with jsdoc -c jsdoc.json, run this in the root project directory
  • Note: include @module <name> in a JSDoc comment at the top of the file, for any functions in the file, include @memberof module:<name>, see DrinkScreen.js for an example.
